Git常用命令

[TOC]

一、安装与配置

  1. 查看版本

    git --version

  2. 查看帮助

    git --help

  3. 配置开发者用户名（全局）

    git config --global user.name 'youyuanbo'

  4. 配置邮箱（全局）

    git config --global user.email 'youyuanbo.zh@qq.com'

  5. 查看当前配置

    git config --list

二、初始化或克隆项目

  1. 初始化项目

    进入到项目的根目录，输入以下命令

    git init

  2. 克隆项目

    在克隆项目时，如果使用SSH通信，需要先进行SSH配置；如果使用http通信，需要输入用户名与密码

    git clone git@github.com:youyuanbo/note.git

三、本地基本命令

  1. 将文件目录添加到暂存区

    添加指定目录或文件

    git add XXX

    添加所有文件

    git add .

  2. 将文件提交到版本库

    git commit -m "注释信息"

  3. 同时将文件添加到索引与版本库

    使用此命令的前提是文件已经是 tracked状态

    git commit -am "注释信息"
git commit -a -m "注释信息"

  4. 查看本地文件状态

    git status

  5. 查看文件版本前后的区别

    +表示添加的代码；-表示删除的代码

    • 查看工作目录和暂存区的区别

      git diff XXX

    • 查看工作目录和本地仓库的区别

      git diff HEAD

    • 查看暂存区和本地仓库的区别

      git diff -cached

  6. 查看文件的提交历史

    git log XXX

  7. 删除文件

    git rm XXX

  8. 撤销修改

    • 修改文件后，没有提交到暂存区（未add，未commit）

      git checkout XXX

    • 修改文件后，已经提交到暂存区，没有提交到版本库（已经add，没有commit）

      git reset HEAD XXX
git checkout XXX

    • 修改文件后，已经提交到版本库（已经add，已经commit）

      git reset 版本标识符
git status

      再参考前两种情况

  9. 删除文件

    从工作树和index中删除文件

    git rm [-f | --force] [-n] [-r] [--cached] [--ignore-unmatch] [--quiet] [--] <file>…
git rm XXX

四、分支与合并

  1. 查看当前仓库中可用的分支

    git branch

    返回结果中，分支前面标有“*”的是当前分支

  2. 创建新的分支

    创建新分支dev

    git branch dev

  3. 切换分支

    • 切换到已经存在的分支

      git checkout dev

    • 切换到不存在的分支（创建并切换分支）

      git checkout -b dev

  4. 在master分支下，合并其他分支

    git checkout master
git merge dev

  5. 将本地分支推送到远程

    • 第一次推送

      git push -u origin master
git push -u origin dev

    • 非第一次推送

      git push origin master
git push origin dev

  6. 删除分支

    • 删除本地分支

      git branch -d dev

    • 删除远程分支

      直接删除

      git push origin --delete dev

      推送一个空的分支

  7. bug分支

    临时挂起当前分支，去解决其他问题

    1. 暂存当前分支

      git status

    2. 查看暂存挂起的工作区

      git stash list

    3. 回复挂起的工作区

      分布完成：

      1. 恢复

        git stash apply

      2. 清除

        git stash drop

      3. 同时恢复并清除

        git stach pop

  8. 标签管理

    1. 设置标签

      git tag v1.0

    2. 查看所有标签

      git tag

    3. 将本地标签推送到服务器上

      • 单个

        git push origin v1.0

      • 批量

        git push origin --tags

    4. 删除标签

      • 本地

        git tag -d v1.0

      • 远程

        git push origin:refs/tags/v1.0

五、分享与更新项目

  1. 配置远程仓库

    1. 在cmd命令行中执行命令，生成公钥与私钥

      ssh-keygen -t rsa -C "ug.linux.python@gmail.com"

    2. 将公钥配置到github或者gitee上

      公钥存放在：C:/user/XXX/.ssh

  2. 添加远程仓库路径

    git remote add origin "XXX"

  3. 将本地仓库推送到远程

    git push -u origin master
git push origin master

  4. 同步远程仓库到本地

    git pull
